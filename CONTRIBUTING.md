# 贡献指南 · CONTRIBUTING

感谢你愿意为本友链仓库做出贡献！请在提交前阅读以下规范。

---

## 📌 申请友链前的自检清单

在提交申请之前，请确保你的站点满足以下条件：

- [ ] 站点可以正常访问（非 localhost / 局域网地址）
- [ ] 站点使用 `https://` 协议（安全连接）
- [ ] 站点内容健康，无违法、色情、赌博等不良信息
- [ ] 站点有实质性内容，非空白页或未完成的占位页
- [ ] 站点名称、描述等信息填写完整且真实

---

## ✏️ PR 规范

### Commit Message 格式

```
feat: 添加友链 - [站点名称]
fix:  修正友链 - [站点名称]（如 URL 更新、描述修改）
del:  移除友链 - [站点名称]（附原因）
```

### PR 标题格式

```
[友链申请] 站点名称
[友链修正] 站点名称 - 修改内容简述
```

### PR 描述模板

提交 PR 时请在描述中附上以下信息：

```markdown
## 站点信息

- **站点名称**：
- **站点地址**：
- **头像地址**：
- **站点简介**：
- **分类标签**：

## 自检

- [ ] 站点可正常访问
- [ ] 使用 HTTPS
- [ ] 内容健康合规
- [ ] JSON 格式正确（已验证）
- [ ] id 不与现有条目重复
```

---

## 🔍 JSON 验证方法

提交前请确认 `links.json` 格式正确：

**方式 1：** 粘贴到 [jsonlint.com](https://jsonlint.com) 在线验证

**方式 2：** 命令行验证
```bash
# Node.js
node -e "JSON.parse(require('fs').readFileSync('links.json','utf8'));console.log('✅ JSON 有效')"

# Python
python -c "import json; json.load(open('links.json')); print('✅ JSON 有效')"
```

## ❓ 常见问题

**Q：头像图片挂了怎么办？**  
A：页面会自动降级显示 🌐 图标，建议使用 CDN 或将头像托管到稳定的地址。

**Q：我的站点是 HTTP 的，可以申请吗？**  
A：暂不接受纯 HTTP 站点，请先为站点配置 HTTPS。

**Q：可以修改已提交的友链信息吗？**  
A：可以，提交 PR 修改对应条目，并在 PR 标题注明 `[友链修正]`。

**Q：可以同时申请多个站点吗？**  
A：原则上每位作者只收录一个主站。如有特殊情况请在 Issue 中说明。
